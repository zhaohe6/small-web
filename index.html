c++学习1

问题一：include头文件的作用

解释：#include<>形如这样的形式，include的作用就是把尖括号的内容引入到.cpp文件中，这就是预编译的过程，之后用这个合成的文件来进行编译。当我们使用#include<iostream.h>时，就不用using namespace std;了，因为已经包含了。但是使用没有.h的头文件时，就必须加上using namespace std;

问题二：c++的命名空间—为什么总是要写using namespace std; ???

解释：C++标准程序库中的所有标识符（为函数或者变量等起的名字）都被定义在一个名为std的namespace中。由于namespace的概念，使用C++标准程序库的任何标识符时都有三种选择：

  1. 直接指定表示符

    例如：std::iostream而不是iostream.

    完整语句为：std::cout<<std::hex<<3.4<<std::endl;//hex表示是以十六进制的形式输出3.4

  2. 使用using关键字

    加入：using std::cout; using std::endl; using std::cin;则前面 的语句可以改写成：

    cout<<std::hex<<3.4<<endl;

  3. 使用 using namespace std;

    这样std命名空间内定义的所有标识符都有效。就好像把他们声明为全局变量一样，那么以上语句又可以改为：cout<<hex<<3.4<<endl;

    当使用的时候，该头文件没有定义全局命名空间，必须使用C++所规定的标准的命名空间（即：namespace std 这就告诉编译器我要使用std命名空间中的标识符了），这样才能正确使用cout、endl等功能。

问题三：C++的指针数组和指向数组的指针傻傻分不清？？

​ 解释：指针数组，注意中心词：“数组”，这就说明，这是一个数组，但是为啥又有 “指针” 二字呢？ 这是因为这个数组里面放的是指针，就是这样！指向数组的指针，中心词是指针，就说明这是一个指针，只不过这个指针是指向数组的。

Springboot学习笔记

@PropertySource and @ImportResource

@PropertySource加载指定的类路径下的配置文件，这个配置文件不一定是.yml或者.properties文件，可以是自己定义的在recourse文件夹下的文件；@PropertySource（value={“classpath=persion.properties”}）这里的persion.properties就是一个自定义的配置文件。

@ImportResource注解用在主类中，来导入一个自定义的xml文件，ImportResource（location= {“classpath=beans.xml”}）其中beans.xml就是一个自定义的xml文件。但是一般不推荐用自定义的xml文件，而是建议使用配置类的方式，在类的上面标注@configration注解就行，在方法上标注@bean注解说明这个方法的返回值添加倒容器中，容器中这个组件的默认id就是方法名。

Thtmeleaf依赖

简单说，Thymeleaf 是一个跟 Velocity、FreeMarker 类似的模板引擎，它可以完全替代 JSP 。 相较与其他的模板引擎，它有如下三个极吸引人的特点：

  • 1.Thymeleaf 在有网络和无网络的环境下皆可运行，即它可以让美工在浏览器查看页面的静态效果，也可以让程序员在服务器查看带数据的动态页面效果。这是由于它支持 html 原型，然后在 html 标签里增加额外的属性来达到模板+数据的展示方式。浏览器解释 html 时会忽略未定义的标签属性，所以 Thymeleaf 的模板可以静态地运行；当有数据返回到页面时，Thymeleaf 标签会动态地替换掉静态内容，使页面动态显示。
  • 2.Thymeleaf 开箱即用的特性。它提供标准和 Spring 标准两种方言，可以直接套用模板实现 JSTL、 OGNL表达式效果，避免每天套模板、改 Jstl、改标签的困扰。同时开发人员也可以扩展和创建自定义的方言。
  • 3.Thymeleaf 提供 Spring 标准方言和一个与 SpringMVC 完美集成的可选模块，可以快速的实现表单绑定、属性编辑器、国际化等功能。

JSP介绍

JSP（全称JavaServer Pages）是由[Sun Microsystems](https://baike.baidu.com/item/Sun Microsystems)公司主导创建的一种动态网页技术标准。JSP部署于网络服务器上，可以响应客户端发送的请求，并根据请求内容动态地生成HTML、XML或其他格式文档的Web网页，然后返回给请求者。JSP技术以Java语言作为脚本语言，为用户的HTTP请求提供服务，并能与服务器上的其它Java程序共同处理复杂的业务需求。

1.get请求的会将提交的数据显示在url的后面，post请求不会，所以post请求的更安全一些

js进阶

变量对象

在局部作用域中，当函数执行完毕，局不变量就没有存在的必要了，因此垃圾回收器很容易做出判断并回收，但是全局变量什么时候释放没存很难却定，因此在开发中尽量避免使用全局变量。

执行上下文

执行上下文可以理解为当前代码的执行环境，他会形成一个作用域，在执行代码的过程中会创建执行上下文。

函数调用栈

在js中调用函数的行为使引擎寻求帮助，这个帮助来自引擎的朋友——函数调用栈。当一个函数被调用时，这个函数会出现在函数调用栈中，这个函数的执行上下文（局部执行上下文），也会在全局执行上下文中出现

函数调用栈、执行上下文、和全局作用域是JS引擎的三个重要部分。
