问题一：include头文件的作用

解释：#include<>形如这样的形式，include的作用就是把尖括号的内容引入到.cpp文件中，这就是预编译的过程，之后用这个合成的文件来进行编译。当我们使用#include<iostream.h>时，就不用using namespace std;了，因为已经包含了。但是使用没有.h的头文件时，就必须加上using namespace std;

问题二：c++的命名空间—为什么总是要写using namespace std; ???

解释：C++标准程序库中的所有标识符（为函数或者变量等起的名字）都被定义在一个名为std的namespace中。由于namespace的概念，使用C++标准程序库的任何标识符时都有三种选择：

  1. 直接指定表示符

    例如：std::iostream而不是iostream.

    完整语句为：std::cout<<std::hex<<3.4<<std::endl;//hex表示是以十六进制的形式输出3.4

  2. 使用using关键字

    加入：using std::cout; using std::endl; using std::cin;则前面 的语句可以改写成：

    cout<<std::hex<<3.4<<endl;

  3. 使用 using namespace std;

    这样std命名空间内定义的所有标识符都有效。就好像把他们声明为全局变量一样，那么以上语句又可以改为：cout<<hex<<3.4<<endl;

    当使用的时候，该头文件没有定义全局命名空间，必须使用C++所规定的标准的命名空间（即：namespace std 这就告诉编译器我要使用std命名空间中的标识符了），这样才能正确使用cout、endl等功能。

问题三：C++的指针数组和指向数组的指针傻傻分不清？？

​ 解释：指针数组，注意中心词：“数组”，这就说明，这是一个数组，但是为啥又有 “指针” 二字呢？ 这是因为这个数组里面放的是指针，就是这样！指向数组的指针，中心词是指针，就说明这是一个指针，只不过这个指针是指向数组的。

指针数组的定义：int*a[4];

数组指针的定义：int(*a)[4];
